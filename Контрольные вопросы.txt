1. Какие профессиональные компетенции формируются в ПМ.02?ПМ.02 «Разработка и администрирование баз данных» направлен на формирование компетенций, необходимых для работы с данными. Ключевые из них:

Проектирование: умение разрабатывать структуру базы данных (создание таблиц, определение полей, типов данных), строить логическую модель предметной области.
Реализация: написание SQL-запросов для создания БД (CREATE), изменения структуры (ALTER), наполнения данными (INSERT) и выборки информации (SELECT, JOIN, агрегатные функции).
Администрирование: управление доступом к данным, резервное копирование и восстановление, оптимизация производительности запросов.
Интеграция: разработка приложений на языке Python, которые взаимодействуют с базой данных MySQL через соответствующие библиотеки.
2. Почему выбран стек: Python + MySQL + tkinter? Какие альтернативы могли быть?Этот стек является классическим решением для изучения основ разработки прикладного ПО с использованием баз данных по нескольким причинам:

Python: простой в изучении язык с понятным синтаксисом, огромным сообществом и множеством библиотек для работы практически со всем.
MySQL: одна из самых популярных реляционных СУБД в мире. Она бесплатна, надежна, хорошо документирована и отлично подходит для обучения принципам работы с БД.
tkinter: стандартная библиотека Python для создания графических интерфейсов (GUI). Её главное преимущество — не требует отдельной установки, так как входит в стандартную поставку Python, что идеально для учебных целей.

Для языка программирования: JavaScript (с Node.js), Java, C# (.NET).

Для GUI-библиотеки: PyQt / PySide (мощный фреймворк), Kivy (для мобильных и мультитач-приложений), wxPython.
3. Что такое ER-диаграмма? Для чего она нужна?ER-диаграмма (Entity-Relationship Diagram) — это диаграмма сущность-связь. Это инструмент концептуального проектирования базы данных, который визуализирует её структуру до написания кода.Она состоит из трех основных элементов:

Сущности (Entities): объекты реального мира, о которых хранятся данные (например, Студент, Книга).
Атрибуты (Attributes): свойства сущностей (у Студента это могут быть ФИО, Номер зачетки).
Связи (Relationships): ассоциации между сущностями (студент берет книгу).
Назначение: ER-диаграмма помогает наглядно спроектировать модель данных, выявить недостающие сущности или атрибуты и определить типы связей между ними ещё до начала реализации, что предотвращает ошибки на этапе кодирования.

4. Какие типы связей между таблицами существуют? Приведите примеры из своего варианта.В реляционных базах данных выделяют три основных типа связей:

Один-к-одному (1:1): одной записи в таблице А соответствует ровно одна запись в таблице Б. Пример: таблица Пользователь и таблица Профиль_пользователя. У каждого пользователя может быть только один расширенный профиль.
Один-ко-многим (1:N): одной записи в таблице А может соответствовать несколько записей в таблице Б. Самый частый тип связи. Пример: таблица Автор и таблица Книги. Один автор может написать много книг (Author_ID в таблице Книги будет внешним ключом).
Многие-ко-многим (M:N): одной записи в таблице А может соответствовать несколько записей таблицы Б, и наоборот. Реализуется через третью, связующую таблицу. Пример: таблица Студенты и таблица Курсы. Студент может посещать много курсов, а на курсе может учиться много студентов. Связующая таблица Запись_на_курс будет содержать пары ID_Студента и ID_Курса.
5. Что такое Git? Чем отличается git commit от git push?Git — это распределенная система контроля версий. Она позволяет отслеживать изменения в файлах проекта, возвращаться к предыдущим версиям, работать над проектом совместно с другими разработчиками, не боясь потерять данные или перезаписать чужую работу.

Разница между командами:

git commit: эта команда сохраняет снимок текущего состояния вашего проекта в локальном репозитории. Вы фиксируете изменения у себя на компьютере. Это похоже на сохранение файла в текстовом редакторе, но с сохранением истории изменений и комментарием к ним.
git push: эта команда отправляет ваши зафиксированные коммиты из локального репозитория в удаленный (например, на GitHub, GitLab или Bitbucket). Это нужно, чтобы поделиться своей работой с командой или сделать её резервную копию.
Проще говоря: commit — сохранить у себя, push — отправить другим.

6. Как проверить, что Python установлен и tkinter работает?Самый надежный способ — создать небольшой скрипт и запустить его.

Откройте терминал (командную строку).
Проверьте версию Python командой python --version или python3 --version.
Создайте файл, например test_tk.py, и вставьте в него следующий код:
python
Копировать
import tkinter as tk

root = tk.Tk()
label = tk.Label(root, text="Tkinter работает!")
label.pack()
button = tk.Button(root, text="Закрыть", command=root.destroy)
button.pack()
root.mainloop()
Запустите скрипт командой python test_tk.py (или python3 test_tk.py).Если откроется небольшое окно с надписью "Tkinter работает!" и кнопкой "Закрыть" — значит, обе технологии установлены и настроены корректно.
7. Какие меры безопасности нужно соблюдать при работе за компьютером?Основные правила цифровой гигиены:

Пароли: использовать сложные, уникальные пароли для разных сервисов. По возможности включать двухфакторную аутентификацию (2FA).
Программное обеспечение: регулярно обновлять операционную систему, браузер и антивирус. Не устанавливать программы из непроверенных источников.
Фишинг: не переходить по подозрительным ссылкам в письмах и сообщениях. Проверять адреса сайтов перед вводом учетных данных.
Данные: не публиковать личную информацию (адрес, телефон, паспортные данные) в открытом доступе. Быть осторожным с тем, что вы загружаете в интернет.
Физическая безопасность: блокировать компьютер (Win+L), когда отходите от рабочего места.
8. Что означает IP-адрес 127.0.0.1?IP-адрес 127.0.0.1 — это специальный адрес, называемый localhost (локальный хост). Он представляет собой "петлю обратной связи" (loopback address). Любой сетевой трафик, отправленный на этот адрес, не уходит во внешнюю сеть, а немедленно возвращается обратно на тот же самый компьютер.Это используется для тестирования сетевых приложений. Например, если ваша программа-сервер запущена на вашем же компьютере, клиентская часть может подключаться к ней по адресу 127.0.0.1, не требуя подключения к реальной сети.

9. Как добавить файл в репозиторий Git (последовательность команд)?Стандартный рабочий процесс выглядит так:

Инициализация репозитория (если он еще не создан): git init
Проверка статуса, чтобы увидеть неотслеживаемые файлы: git status
Добавление файлов в область подготовленных изменений (индекс). Чтобы добавить конкретный файл: git add имя_файла. Чтобы добавить все новые и измененные файлы: git add .
Фиксация изменений в локальной истории: git commit -m "Описание сделанных изменений"
Отправка изменений в удаленный репозиторий (если требуется): git push origin название_ветки
10. Зачем нужен файл README.md в проекте?README.md — это главный информационный файл проекта, расположенный в его корневой папке. Его задача — быть "витриной" вашего проекта и первым источником информации для любого человека (включая вас самих в будущем).Обычно он содержит:

Название и краткое описание проекта.
Инструкции по установке и запуску (какие зависимости нужны, какие команды выполнить).
Примеры использования.
Сведения об авторах и лицензии.Наличие хорошего README.md показывает профессионализм разработчика и значительно упрощает использование и поддержку проекта.
